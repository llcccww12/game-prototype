## 电梯演讲

一款像素武侠风格的 Roguelike 横版动作游戏。玩家扮演一名流浪剑客，在随机生成的房间中战斗，每局收集技能和被动，Build 搭配决定胜负，约 10 分钟一局，可随时重开。

**一句话**：「随机江湖，一剑一局。」

---

## 核心愿景

玩家在黑暗的武侠世界中战斗，通过随机获得的技能和被动组合出独特 Build，挑战 Boss 并追求最佳 Build 组合。游戏的核心乐趣在于**Build 探索**（这次出什么技能？）和**战斗执行**（操作能不能打出来？），两者缺一不可。

---

## 核心循环

### 30秒循环（战斗）

```
敌人出现 → 移动躲避 → 普攻 / 闪避 → 技能释放 → 击杀 → 捡技能/被动
```

战斗以 2-3 秒为一轮，玩家在移动、普攻、技能三个动作间做决策。

### 5分钟循环（房间）

```
进入房间 → 观察敌人布局 → 选择入场策略 → 清场 → 选择路线 → 进入下一房间
```

每 3-4 个房间会遇到一个强化敌人或精英。

### 单局循环（10-15分钟）

```
选难度 → 3个普通房间 + 1个Boss房 → 死亡或通关 → 查看本局Build → 重开或收工
```

Roguelike 核心：死亡不是失败，每次死亡都是信息积累。

### 局外循环（长期）

```
解锁新初始武器 → 解锁新技能 → 研究新Build → 挑战更高难度
```

## 视觉身份锚点

### 美术风格

**像素卡通暗黑风** — 参考暖雪：暗色背景（深灰/墨色）+ 高饱和度角色/特效点缀，对比强烈，视觉层次分明。

### 色调方案

**冷色系（蓝紫）为主** — 背景和敌人偏冷调蓝紫，玩家角色和技能特效用暖色（橙红）形成对比。血液/击杀效果用亮色。

### UI 设计

**暖雪式 HUD** — 底部技能槽显示当前技能，顶部血条 + 当前房间编号。简洁但信息完整，不遮挡战斗视野。

### 命名（暂定）

游戏代号：**霜刃**（Snowblade）

## 游戏系统概述

### 战斗系统

- **移动**：虚拟摇杆控制，8方向
- **普攻**：初始武器，默认近战刀剑，三段连击
- **闪避**：消耗耐力，向移动方向快速位移，无敌帧
- **技能**：战斗中可拾取，替换当前技能槽
- **被动**：战斗中可拾取，最多携带3个，叠加生效

### 技能系统

**初始角色**：自带1个技能（主动或被动，随机分配）

**上限**：3主动 + 3被动 = 最多6个

**获取**：每清1个房间出现技能选择界面（三选一）

| 技能名称 | 类型 | 效果 |
|---------|------|------|
| 剑气斩 | 主动-近战 | 前方扇形范围斩击 |
| 飞燕回翔 | 主动-位移 | 突进一段距离，可穿敌 |
| 冰魄掌 | 主动-远程 | 发射冰球，击退敌人 |
| 御风步 | 主动-位移 | 短距离瞬移，附带无敌帧 |
| 破军 | 主动-爆发 | 短暂蓄力后全屏伤害 |

初始武器提供1个技能（随机），其余靠闯关获得。

### 被动系统

| 被动名称 | 效果 |
|---------|------|
| 血战 | 生命越低，伤害越高 |
| 铁壁 | 受到伤害减少10% |
| 连斩 | 连续击杀敌人后攻速提升 |
| 吸血 | 造成伤害的5%转化为生命 |
| 疾风 | 移动速度提升15% |

每局可拾取最多3个被动，随机掉落。

### 地图系统

- **房间结构**：3个普通房间 + 1个Boss房，共4个房间
- **随机性**：房间内敌人类型和数量随机，路线选择随机
- **房间内容**：普通敌人（2-3种）、精英敌人（固定Boss房）

### 敌人类型

| 敌人 | 行为 |
|------|------|
| 小兵 | 简单近战追击 |
| 射手 | 远程攻击，需要躲避 |
| Boss | 多阶段，有特定招式 |

### 难度系统

- 单难度（V1），后续可扩展为多难度

---

## 数值设计原则

- **玩家血量**：100，初始无回复，靠技能/被动获得回复
- **普攻伤害**：20（可被被动提升）
- **敌人血量**：小兵30，Boss 300
- **耐力**：100，每2秒回复10，闪避消耗20

---

## MVP 定义（版本1）

**目标**：验证核心循环是否好玩

包含内容：
- 1把初始武器（刀剑，三段普攻）+ 1个初始技能（随机）
- 5个主动技能（剑气斩、飞燕回翔、冰魄掌、御风步、破军）
- 5个被动（血战、铁壁、连斩、吸血、疾风）
- 2种小兵敌人
- 1个Boss（单阶段）
- 3个随机普通房间 + 1个Boss房
- 每清1个房间出现技能选择（三选一）
- 触控虚拟摇杆
- Android 打包测试

**不包括**：
- 局外解锁系统
- 多难度
- 剧情/叙事
- 音效/音乐（可后续添加）

## 风险评估

| 风险 | 等级 | 应对 |
|------|------|------|
| Godot 手机触控体验不流畅 | 中 | 先做原型验证手感 |
| Build 组合不平衡 | 中 | 数值先保守，后续调参 |
| 像素美术工作量 | 高 | 使用 Kenney 素材或 AI 生成，聚焦核心循环 |
| 手机性能 | 低 | Godot 导出优化成熟 |
