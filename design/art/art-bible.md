## 1. Visual Identity Statement

### One-Line Visual Rule
**暗色世界，亮色英雄** — 玩家角色和技能特效是画面中最亮的存在，敌人次之，背景最暗。

### Supporting Principles

1. **对比驱动注意力**：玩家永远是最亮的，敌人次亮，背景压暗。视觉焦点不需要学习，玩家本能看向最亮的元素。
   *设计测试：当战斗混乱时，玩家能否在 0.3 秒内找到自己的角色？*

2. **像素但不粗糙**：16x16 到 32x32 基础像素，角色轮廓清晰，无歧义。细节用像素堆叠而非模糊。
   *设计测试：角色缩小到 32x32 像素后，是否仍能一眼辨认？*

3. **色彩即信息**：每个颜色有固定语义，不混用。玩家见到某种颜色就知道含义。
   *设计测试：把游戏灰度化后，主要信息是否仍然可读？*

## 2. Mood & Atmosphere

| 游戏状态 | 情绪目标 | 光照特征 | 氛围形容词 | 能量等级 |
|---------|---------|---------|-----------|---------|
| **普通房间** | 紧张但不压迫 | 低对比冷光（蓝紫），房间中心略亮 | 寂静、暗藏杀机 | 测量/警觉 |
| **精英/Boss房间** | 高压、危险 | 高对比，暖色威胁光从敌人方向投射 | 压迫感、决断时刻 | 激烈 |
| **技能选择界面** | 掌控感、期待 | 中性亮度，选项发光突出 | 喘息、策略暂停 | 平静/专注 |
| **死亡** | 惋惜但不沮丧 | 全屏冷色渐暗，红色警示 | 遗憾中带着再来一局冲动 | 沉寂 |
| **通关** | 满足、成就感 | 暖光从屏幕中心向外扩散 | 释放感、值得重玩 | 释放 |

## 3. Shape Language

### 角色剪影
- **玩家**：竖长身形（剑客），肩宽头小，快速动作感轮廓。剑的形状是核心识别特征。
- **敌人**：横向宽大或不对称（刺客型），比玩家更"宽"以增加威胁感。
- **Boss**：占据画面 1/4 以上，大轮廓有明确攻击方向指示。

### 环境几何
- **主导**：硬直角 + 斜面（武侠建筑感）
- **装饰**：少量垂直线条（竹林/旗帜意向）
- **地面**：水平线明确，平台边缘锐利

### UI 形状语法
- 技能槽：圆角矩形，带发光边框（激活态 vs 冷却态）
- 血条：线性，左→右递减，颜色从绿→黄→红过渡
- 图标：像素风格，2x2cm 可读

### 主形状 vs 辅助形状
- **吸引眼球**：玩家、技能特效、Boss — 高饱和度 + 动态
- **环境背景**：平台、装饰物 — 低饱和度，静止

## 4. Color System

### 主色板（世界色调）

| 色值（暂定） | 名称 | 语义角色 |
|------------|------|---------|
| `#1a1a2e` | 墨夜 | 主背景，深沉武侠氛围 |
| `#16213e` | 暗渊 | 次级背景，层次分离 |
| `#0f3460` | 冷蓝 | 房间/平台基色，稳定感 |
| `#e94560` | 血红 | 伤害、危险、Boss指示 |
| `#f5a623` | 烈阳 | 玩家角色主色、技能特效 |
| `#7b2cbf` | 幽紫 | 敌人主色、诅咒/负面效果 |
| `#eaeaea` | 霜白 | 高光、文字、重要提示 |

### 语义颜色规则
- **橙色/金色** = 玩家主动技能、特效、增益
- **红色** = 伤害来源、敌人攻击、危险区域
- **紫色** = 敌人角色、减益效果
- **绿色** = 治疗、生命恢复、增益
- **白色** = 文字、高亮、重要提示
- **灰色** = 背景装饰、不可交互物体

### UI 调色板
- HUD 背景：`#000000` @ 60% 透明度
- 技能冷却：灰化 + 白色倒计时数字
- 血条：满血 `#4ade80` → 50% `#fbbf24` → 25% `#ef4444`

### 色盲安全
- 伤害警告：红色 + 屏幕闪烁（双重信号）
- 增益/减益：颜色 + 图标形状双重标识
- 敌人危险：紫色 + 闪烁预警

## 5. Character Design Direction

### 玩家角色
- **体型**：16x16 或 32x32 像素基础，高度占 2/3
- **服装**：暗色主调 + 橙色点缀（腰带/剑穗），剑是视觉中心
- **动画帧数**：移动 4 帧，攻击 3 帧，闪避 2 帧
- **LOD 原则**：32x32 内保留轮廓和配色信息

### 敌人设计
- **小兵A（刀客）**：近战，紫色衣服，横向宽大轮廓
- **小兵B（弓手）**：远程，紫色衣服，弓箭是识别特征
- **Boss**：体型大（64x64 或更大），红色为主，轮廓有攻击方向预兆

### 视觉区分原则
- 玩家 vs 敌人：配色差异（暖 vs 冷）超过轮廓差异
- 近战 vs 远程敌人：从轮廓高度区分（横向 vs 纵向）

## 6. Environment Design Language

### 建筑风格
- 武侠客栈/废墟风格，倾斜屋顶暗示
- 背景：深色垂直结构（柱子/竹林）提供深度
- 前景平台：实心矩形，墨色填充 + 冷蓝描边

### 纹理哲学
- **无 PBR 贴图**：纯色 + 像素噪点
- **地面**：纯色 + 1px 高光边（顶部）
- **墙壁**：渐变深色（顶部深 → 底部略浅）

### 道具密度
- **战斗区域**：极简，无装饰物干扰
- **过渡区域**：允许 1-2 个装饰道具（旗帜、灯笼）
- **Boss 房间**：环境更复杂（柱子在边缘，不遮挡战斗区域）

### 环境叙事
- 地面裂缝 = 危险
- 发光物体 = 可交互/可拾取
- 暗处敌人 = 需要警惕

## 7. UI/HUD Visual Direction

### HUD 布局
```
[房间进度: ██░░]  [血条 ████████░░]  [???]

   [敌人/环境]

[摇杆]                    [技能1][技能2][技能3]
```

- **顶部**：血条（满宽60%），房间进度（4格）
- **底部左**：虚拟摇杆（半透明）
- **底部右**：3个技能槽（激活/冷却态区分）
- **被动显示**：血条下方小图标，3个槽位

### 字体
- **数字/状态**：等宽像素字体，高对比度
- **标题**：粗体像素字，游戏感强

### 图标风格
- 像素风格，16x16，1px 描边
- 主动技能：橙色边框
- 被动：白色边框 + 图标

### 动画
- 技能冷却：顺时针填充动画
- 伤害数字：向上飘出 + 淡出
- 拾取技能：发光放大 + 缩小确认

## 8. Asset Standards

### 文件格式
- **精灵图**：PNG（背景透明）
- **动画**：SpriteSheet（行=帧，列=动作）
- **背景**：PNG 或 Godot TileMap

### 分辨率规格
| 资产类型 | 最大尺寸 | 备注 |
|---------|---------|------|
| 角色精灵 | 64x64 | 站立/攻击/移动/受伤 |
| 敌人精灵 | 64x64 | 含方向变体 |
| 技能特效 | 128x128 | 带透明通道 |
| 背景图层 | 1920x1080 | 适配手机竖屏 |
| UI 图标 | 32x32 | 导出 2x（64x64 显示） |

### 命名规范
```
[category]_[name]_[state].png
player_idle_01.png
enemy_sword_walk_01.png
skill_fire_impact_01.png
```

### 导出设置
- Godot Import：Filter → Nearest（像素锐利）
- 禁用 Mipmap（像素游戏不需要）
- 纹理压缩：VRAM 移动端优先

## 9. Style Prohibitions

### 绝对禁止
- **渐变滥用**：背景不用大范围渐变，保持色块感
- **模糊/柔边**：所有资产必须是锐利像素
- **无对比 UI**：HUD 元素必须有明确边界，不融入背景
- **彩色剪影**：敌人剪影不能与背景同色（必须有轮廓或色差）

### 避免
- 3D 渲染风格（保持 2D 像素）
- 过度细节（像素级别控制在 32x32 内可读）
- 暖色调敌人（敌人必须与玩家有明显的冷/暖色对比）
- 复杂 UI 动画（战斗 UI 必须快、清晰、无延迟感）

## Reference Direction

### 1. 暖雪 (Warm Snow)
- **取**：暗色背景 + 高饱和度角色对比，暗黑武侠氛围，像素卡通风格
- **避**：暖雪的水墨 UI 风格（我们用像素 HUD）
- **来源**：`https://store.steampowered.com/app/1978750`

### 2. Dead Cells
- **取**：像素精灵的流畅动画感，技能特效的明亮色彩
- **避**：3D 视角（我们是固定横版）
- **来源**：`https://store.steampowered.com/app/588650`

### 3. 空洞骑士 (Hollow Knight)
- **取**：冷暖色对比，蓝紫色调环境，清晰剪影设计
- **避**：复杂的场景道具密度
- **来源**：`https://store.steampowered.com/app/367520`

### 4. 以撒的结合 (The Binding of Isaac)
- **取**：HUD 简洁设计，技能选择界面的清晰三选一
- **避**：宗教主题和暗黑色调过度
- **来源**：`https://store.steampowered.com/app/113200`
